# 《斗篮机》街机球馆 PK 版设计规格

## 背景与审核目标

本次重设计用于回应提案评估反馈：“小玩法设计过于简陋”“美术效果不足”“主播与观众互动性弱”“玩法核心内容存在穿帮、违和、简陋”。新版不改变已确认的合规规则，只升级玩法包装、互动节奏、视觉反馈和提案可展示内容。

核心目标：

- 让玩法从“投篮面板”升级为“直播街机篮球赛事舞台”。
- 在视频和交互稿中完整体现从开场、入队、投篮、礼物技能到结算的流程。
- 强化观众参与感，让点赞、礼物、入队、榜单、MVP 都有可见反馈。
- 避免赌博或概率误解：点赞固定 30% 命中，礼物按官方弹幕小玩法礼物分档固定命中和固定得分，不由主播配置命中率。

## 保留规则

- 观众可选择红队或蓝队，入队后不可换队。
- 点赞触发普通投篮，固定 30% 命中率，命中加 1 分。
- 礼物触发技能投篮，按官方弹幕小玩法礼物分为低价、中价、高价三档。
- 低价礼物命中加 1 分，中价礼物命中加 2 分，高价礼物命中加 3 分。
- 主播可设置对局时长和队伍口令；时长支持 3、5、10、15、20、25、30 分钟。
- 对局结束展示胜负、MVP、红蓝队贡献榜前五。

## 新玩法主线

### 1. 开场仪式

主播点击开始后进入 3 秒开场倒计时，而不是立刻开始普通投篮。

表现内容：

- 霓虹球馆灯光从暗到亮。
- 红蓝队灯牌依次点亮。
- 队伍口号从左右两侧滑入。
- 已入队观众头像飞入看台或队伍应援区。
- 中央出现“斗篮机开赛”或“READY / GO”动效。

目的：让审核看到玩法有明确开局流程和赛事包装。

### 2. 比赛主舞台

中央区域改为“霓虹街机篮球机”。

主要元素：

- 上方为街机计分屏：红队比分、倒计时、蓝队比分。
- 中央为酷炫篮板和篮筐，包含 LED 灯带、玻璃高光、能量环、篮网反馈。
- 下方为投篮发射区，篮球根据触发类型改变样式。
- 左右为红蓝应援区，显示队名、在线人数、口令、当前队内 TOP 球员。
- 底部为三段信息条：最近进球、贡献榜前三、本场 MVP。

布局原则：

- 信息区不能遮挡篮球、篮筐和投篮轨迹。
- 长昵称和榜单文本使用跑马灯或截断，不把区域撑高。
- 主播操作区与观众视角互不影响，实时流水不能顶长主舞台。

### 3. 普通点赞投篮

点赞触发普通投篮。

表现内容：

- 观众头像或昵称短暂出现在投篮起点。
- 篮球使用普通橙色球皮。
- 轨迹为轻量弧线，带少量星点或速度线。
- 命中时篮筐闪光、比分跳字、最近进球更新。
- 未命中时出现打铁火花或篮筐震动，但不夸张。

### 4. 礼物技能投篮

礼物不再只是“换颜色投篮”，而是按档位体现不同技能等级。

低价礼物：

- 定位：快投球。
- 时长：约 1.5 至 1.8 秒。
- 表现：快速星光、轻量粒子、短促命中反馈。
- 得分：命中 +1。

中价礼物：

- 定位：技能球。
- 时长：约 3.5 秒。
- 表现：根据礼物类型出现差异化特效，例如充能电流、镜面反弹、甜甜圈弧线、能量电池加速。
- 得分：命中 +2。

高价礼物：

- 定位：高光球。
- 时长：约 5 秒。
- 表现：全屏高光、镜头轻震、篮筐慢速爆光、队伍欢呼、礼物 icon 强化出场。
- 得分：命中 +3。

### 5. 互动状态反馈

不改变比赛规则，只增加主播与观众可感知的状态反馈。

新增反馈：

- 连中提示：同一队伍连续命中时出现“红队连中 3 球”。
- 反超提示：比分从落后变领先时出现“蓝队反超”。
- 追分提示：落后队伍连续得分时出现“红队追分中”。
- 最后 30 秒：倒计时进入高压样式，灯光和音效节奏加快。
- MVP 刷新：MVP 变化时出现头像光圈和“本场 MVP 更新”提示。

这些状态只基于实际比分和贡献数据展示，不额外增加抽奖、随机奖励或金钱收益表达。

### 6. 结算颁奖

结算从普通弹窗升级为“赛后颁奖台”。

展示内容：

- 胜方队伍或平局结果。
- 红蓝队最终比分。
- MVP 头像、昵称、得分、命中次数。
- 红蓝队贡献榜前五。
- 本场数据摘要：总投篮次数、点赞触发次数、礼物触发次数、最高连中。
- 下一局提示：重新开局后观众需要重新选择阵营。

## 美术方向

整体风格：霓虹街机、直播赛事、夜场球馆。

关键词：

- 深蓝黑背景。
- 红蓝阵营对抗。
- 玻璃篮板、霓虹灯带、LED 计分屏。
- 篮球机机械感、能量轨迹、礼物技能球。
- 高价礼物要有“高光时刻”，但避免过度遮挡玩法核心信息。

## 需要落地的产物

- Vue/Phaser Demo 主界面升级。
- 开场倒计时和比赛阶段状态。
- 点赞、低价、中价、高价礼物的差异化投篮动效。
- 连中、反超、追分、最后 30 秒、MVP 更新提示。
- 结算颁奖台。
- Figma 主界面交互稿至少一张，最好包含主比赛界面和结算界面。
- 提案评估文档中同步替换玩法描述、主界面交互稿、礼物特效说明和 Demo 视频截图。
- 重新录制实机完整流程视频。

## 非目标

- 不新增付费抽奖、随机奖励、返利、提现或兑换表达。
- 不让主播配置点赞命中率。
- 不新增观众换队功能。
- 不把礼物描述为概率提升或投注结果。
- 不改变当前低/中/高礼物固定得分规则。

## 验收标准

- 审核人员看主界面截图时能明确识别这是一套完整直播小玩法，而不是普通控制台。
- Demo 视频能连续展示开场、入队、点赞投篮、礼物技能、实时榜单、结算颁奖。
- 不同档位礼物在时长、轨迹、球皮、粒子、篮筐反馈上有明显差异。
- 主播配置区、观众视角区、实时流水区互不撑高或遮挡。
- 文案明确说明点赞固定 30% 命中，礼物为官方弹幕小玩法礼物，按档位固定命中和固定得分。
